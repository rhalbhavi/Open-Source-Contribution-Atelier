# 📌 GitHub Issues

### Before writing any code, work in open source almost always begins inside an **Issue**.

---

## What is an Issue?

An **Issue** is a tracking ticket. It is used to record bugs, suggest new ideas, request documentation improvements, or discuss architectural shifts.

---

### Elements of an Issue

- **Issue Title**: The name of the issue which is set by the developer/repo owner.

- **Issue Number**: A unique, sequential identifier automatically assigned to every issue. Seen next to issue title.

- **🏷️ Labels**: Color-coded tags (like `bug`, `enhancement`, `good first issue`, `documentation`) that categorize the task.

- **👤 Assignee**: The developer who is currently responsible for resolving the task.

- **📍 Milestones**: Target release goals that group related issues together.

> [!NOTE]
