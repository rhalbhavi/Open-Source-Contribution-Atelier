# 📌 Issues

### Before writing any code, work in open source almost always begins inside an **Issue**.

---

## ❓ What is an Issue?

An **Issue** is a tracking ticket. It is used to record bugs, suggest new ideas, request documentation improvements, or discuss architectural shifts.

## 🎯 Elements of an Issue

- **Issue Title**: The name of the issue which is set by the developer/repo owner.

- **Issue Number**: A unique, sequential identifier automatically assigned to every issue. Seen next to issue title.

## 🛠️ How to Raise an Issue

1. Navigate to the Repository: Open GitHub in your browser and go to the project's home page.
2. Open the Issues Tab: Locate the horizontal menu bar under the repository name and click Issues.
3. Click New Issue: Click the green New issue button located on the right side of the screen.
4. Choose a Template: If the project uses templates (e.g., Bug Report or Feature Request), click Get started next to your choice. If not, click Open a blank issue.
5. Add a clear, descriptive title summarizing the core problem (e.g., Bug: Login button fails on Android).
6. Write the description by filling in the text block using GitHub Markdown to structure your explanation.
7. Click Submit at the bottom of the form to publish your issue.

---

> [!NOTE]
> Opening an issue is a contribution! If you find a bug and provide detailed reproduction steps, you are saving the maintainers valuable time.
