# Professional Submissions

To keep open-source repositories healthy, you should avoid common anti-patterns that create noise and overhead for maintainers.

---

### Avoid These Anti-Patterns:

1. **The Hit-and-Run PR**: Creating a pull request and never responding to review comments or feedback.
2. **Spam Pull Requests**: Making trivial edits (like adding a single space or fixing punctuation in a comment) just to increase your GitHub contribution count (e.g. during Hacktoberfest).
3. **Working Without Assignment**: Starting work on a complex issue without checking if someone else is already assigned to it or asking the maintainers first.
4. **Duplicate Issues**: Opening a bug report without searching existing issues first to check if it's already reported.

---

> [!CAUTION]
> Spamming or ignoring contribution guidelines can get your PR tagged as spam, which can lead to your GitHub account being restricted. Always make meaningful contributions!
