# ⚠️ Interactive Step-Through Debugging

### Sometimes code doesn't do what we expect. Instead of just printing variables to the console and guessing where things went wrong, professional developers use **Debuggers**.

---

## A Debugger allows you to:

1. **Pause execution** at specific lines (Breakpoints).

2. **Step through** the code line by line.

3. **Inspect variables** in real time as they change.

4. View the **Call Stack** to see exactly how you arrived at a function.

---

## 🛠️ Hands-On Task

### Visit this website to access a live Interactive JavaScript Debugger Simulator tool:

### [https://javascript-debugger-simulator.netlify.app/](https://javascript-debugger-simulator.netlify.app/)

---

### Steps

1. **Set a Breakpoint**: Click the margin (the space directly left of the line numbers) on line 4 (`total += i`). You should see a red dot appear.

2. Click **Start Debugging**.

3. Watch the code execute until it hits your breakpoint and pauses!

4. Notice the **Variables** panel on the right. It shows `i` and `total`.

5. Click **Step Over** repeatedly and watch how the variables change on each iteration of the loop!

6. Click **Continue** to run the rest of the code automatically.
