# 🌐 What is Open Source? 👨‍💻

### At its core, Open Source describes software whose source code is released under a license that grants anyone the right to inspect, modify, distribute, and use it for any purpose.

Instead of hiding source code inside proprietary boxes, open source projects embrace **transparency** and **collaborative development**.

---

## 🎯 The Open Source Philosophy

Open source is built on the following key principles:

- **🔎 Transparency**: Anyone can review code changes, report problems, and examine the development history/logs.
- **📢 Participation and Contribution**: Diverse contributors from around the world pool their skills to improve software.
- **🤝 Collaboration**: Decisions are often reached asynchronously through issues, discussions, and pull requests.
- **🫶 Inclusivity**: Anyone, from student to expert, can participate and build upon open work. Beginners as well as seasoned developers

---

## ⚔️ Open Source vs. Closed Source

| Feature          | 🔓 Open Source                                  | 🔐 Closed (Proprietary) Source                          |
| :--------------- | :---------------------------------------------- | :--------------------------------------------------- |
| **Source Code**  | Publicly accessible to view and edit            | Locked; only visible to the owner company            |
| **Development**  | Driven by community collaboration               | Driven by company teams and specifications           |
| **Trust**        | Auditable by anyone (security is peer-reviewed) | Security depends entirely on the publisher's promise |
| **Adaptability** | You can fork and customize it                   | Restricted by copyright and end-user agreements      |

---

> [!NOTE]
> Open source doesn't mean _no rules_. It relies on legal **open-source licenses** (like MIT, Apache 2.0, or GPL) to set the guidelines for copying and sharing.
